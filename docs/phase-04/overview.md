# Phase 4 - Clean Architecture and CQRS

## Goal

The goal of Phase 4 is to refactor the project into a layered architecture and introduce CQRS.

---

## What Was Implemented

- separation into Domain, Application, Infrastructure, and API layers
- introduction of MediatR
- implementation of CQRS for the Account module
- refactoring AccountsController to a thin controller
- gradual migration strategy from service-based architecture

---

## Key Concepts

- Clean Architecture
- CQRS (Command Query Responsibility Segregation)
- MediatR pattern
- separation of concerns

---

## Outcome

The project is now modular, maintainable, and ready for advanced features.
