# Phase 5 - Testing

## Goal

The goal of Phase 5 is to add automated tests to MiniLedger.

This phase introduces both unit tests and integration tests to improve confidence in the codebase and make future changes safer.

---

## What Is Added

- test project
- unit tests for CQRS handlers
- unit tests for journal entry validation
- integration tests for authentication
- integration tests for API endpoints
- reusable test infrastructure

---

## Expected Outcome

By the end of Phase 5, MiniLedger should have a reliable automated testing foundation.
