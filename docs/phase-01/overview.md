# Phase 1 - Web API Fundamentals

## Goal

The goal of Phase 1 is to build a fully working ASP.NET Core Web API with a clean architecture and a realistic accounting domain.

This phase focuses on the technical foundation required for building scalable backend systems.

---

## What is MiniLedger?

MiniLedger is an educational backend project inspired by real accounting systems.

It helps developers move from intermediate to advanced backend development by working on a practical domain instead of a trivial example.

---

## What You Will Learn

- ASP.NET Core Web API fundamentals
- Clean project structure
- Entity Framework Core with MySQL
- DTO design and mapping
- Repository pattern
- Service layer design
- Controller design
- Middleware (Exception Handling & Logging)
- Swagger documentation

---

## Core Modules

### Accounts
Represents accounting accounts such as Cash, Bank, Revenue, and Expenses.

### Parties
Represents customers, suppliers, or other related entities.

### Journal Entries
Represents accounting documents.

### Journal Entry Lines
Represents debit and credit lines inside each journal entry.

---

## Expected Outcome

By the end of Phase 1, you will have:

- A working Web API
- Database integration with EF Core
- CRUD for Accounts and Parties
- Journal Entry creation with validation rules
- Middleware for exception handling and logging
- Swagger UI enabled
- Clean and documented GitHub repository
